Java面试笔试之基础知识-1

Java面试笔试之基础知识-1

1、基本概念（上）

1.1、Java语言的优点

  • 纯面向对象（Everything is object，万物皆对象）
  • 平台无关性（一次编译，到处运行）
  • 内置类库（简化程序设计，缩短开发时间）
  • 对web应用开发的支持（Support to Web）
  • 较好的安全性和健壮性（提供了数组边界检测、字节码校验、强类型机制、垃圾回收器、异常处理等）
  • 去除了C++中的难点（头文件、指针、结构、单元、运算符重载）

1.2、Java与C/C++的区别

  • Java为解释型语言，即源代码—>Java编译器编译为字节码—>JVM解释执行。C/C++为编译型语言，执行速度快，但不能够跨平台，而Java执行速度慢，却可以跨平台执行。
  • Java中除基本类型外，所有类型都是类（纯面向对象），不存在全局变量或全局函数。而C语言是面向过程，C++兼具面向对象和面向过程的特点。
  • Java中没有指针，更加安全。
  • Java不支持多重继承，但可以实现多个接口来实现类似效果。
  • Java提供了垃圾回收器实现垃圾的自动回收，而C++需要把释放资源的代码放到析构函数中。
  • Java具有平台无关性，对数据类型分配固定长度，不因平台的差异而产生差异。而C++同一个数据类型在不同平台上的长度不同。
  • Java提供了注释文档的内建支持。
  • Java包含了一些标准库。如JDBC库。

1.3、public static void main（String[] args)方法

这个方法是Java程序的入口方法。其中：

  • public和static没有先后顺序关系。
  • 可以把main()方法定义为final。
  • 可以用synchronized来修饰main()方法。
  • 不能用abstract来修饰main()方法。

不管哪种定义方式，都必须保证main()方法的返回值为void，并有static和public关键字修饰。那么一个.java文件中能否有多个main()方法？

1.4、程序的初始化顺序

1.4.1、初始化的原则

Java程序的初始化一般遵循3个原则（优先级递减）：

  • 静态对象（变量）优先于非静态对象（变量）。
  • 父类优先于子类。
  • 按照成员变量的定义顺序进行初始化。、

1.4.2、初始化的顺序

Java程序初始化工作可以在许多不同共同的代码块中完成，执行的顺序如下：

父类静态变量—>父类静态代码块—>子类静态变量—>子类静态代码块—>父类非静态变量—>父类非静态代码块—>父类构造函数—>子类非静态变量—>子类非静态代码块—>子类构造函数

1.5、Java中的作用域

在Java语言中，作用域是由花括号{}的位置决定的，决定了其定义的变量名的可见性与生命周期。

Java中，变量的类型有3中：成员变量、静态变量、和局部变量。其中，成员变量的作用范围与类的实例化对象的作用范围相同；静态变量不依赖于特定的实例，被所有实例所共享，即只要类被加载，JVM就会给它的静态变量分配空间；而局部变量的作用域是它所在的花括号内。

1.6、构造函数的特点

在Java中，构造函数具有以下特点：

  • 构造函数必须与类的名字相同，且不能有返回值（或void）。
  • 一个类可以有多个构造函数。
  • 构造函数可以有0、1或多个参数。
  • 构造函数总伴着new操作一起调用，且是由系统调用，只运行一次。
  • 主要用于完成对象的初始化工作。
  • 构造函数不能被继承，因此也不能被覆盖，但是可以被重载。
  • 子类可以通过super关键字显示调用父类的构造函数。当父类中没有无参数的构造函数时，子类必须显示调用父类的构造函数。
  • 当父类和子类都没有定义构造函数时，系统会为父类和子类都生成一个默认的无参数的构造函数。
