9.1、Java Collections（集合）框架

Java Collections框架包含了大量集合接口以及这些接口的实现类和操作它们的算法（如排序、查找、反转、替换、复制、取最小/最大元素等），主要提供了List（列表）、Queue（队列）、Set（集合）、Stack（栈）和Map（映射表）等数据结构，其中List、Queue、Set、Stack都继承自Collection接口。
Collection接口是整个集合框架的基础，作用只是提供维护一组对象的基本接口。
下面分别介绍Set、List和Map3个接口：

  • Set表示数学意义上的集合概念。特点是元素不能重复，因此每个元素都实现了equals()方法确保唯一性。有两个实现类：HashSet和TreeSet，TreeSet实现了SortedSet接口，因此是有序的。
  • List又称有序的Collection，按对象进入的顺序保存对象，能对每个元素的插入和删除位置进行精确控制，且对象可以重复。LinkedList、ArrayList和Vector都是实现了该接口。
  • Map提供了一个K-V键值对的数据结构，键唯一，值可以重复。HashMap、TreeMap、LinkedHashMap、WeakHashMap、IdentityHashMap都实现了该接口。其中，HashMap是基于散列表实现的，采用对象的HashCode可以进行快速查询；LinkedHashMap采用列表来维护内部的顺序；TreeMap基于红黑树的数据结构，内部元素按需排列。

简单来说，Java集合框架主要包含两种类型的容器：集合（Collection）和图（Map）。Collection接口又有三种子类型：Set、List和Queue，再下面就是一些抽象类，最后是实现类。常用的主要有：ArrayList、LinkedList、HashSet、LinkedHashSet、HashMap、LinkedHashMap等。
总的来说，Java集合框架是一个用来代表和操纵集合的统一架构，基本包含三部分：接口、实现类和算法。

9.2、迭代器

Iterator（迭代器）是一个对象，用来遍历并选择序列中的对象，可以不了解容器底层的结构就实现对容器的遍历，是一种轻量级容器。在使用时需要注意：

  • 使用容器的iterator()方法返回一个Iterator，然后通过Iterator的next()方法返回第一个元素；
  • 使用Iterator的hasNext()方法判断容器中是否还有元素。若有再使用next()方法返回 下一个元素；
  • 可以通过remove()方法删除迭代器返回的元素。

9.3、ArrayList、Vector和LinkedList的区别

ArrayList、Vector和LinkedList都在java.util包中，均为可伸缩数组，可动态改变长度。

  • ArrayList和Vector都是基于存储元素的Object[]array来实现的，数据内存连续，支持使用下标访问元素，索引数据速度比较快，但是插入数据速度比较慢。它们都有一个初始化容量大小，当这个存储容量满员时，它们就会进行扩充。不同的是，Vector默认扩充为原来的2倍，ArrayList默认扩充为原来的1.5倍。它们的最大区别就是ArrayList所有方法都不同步，而Vector大多数方法是同步的，因此Vector是线程安全，而ArrayList非线程安全，Vector因此性能略逊于ArrayList。
  • LinkedList采用双向列表实现，对数据的索引从列表头开始遍历，因此随机访问效率较低，但是插入数据效率较高，非线程安全。

在实际使用中，当数据的主要操作为索引或只在集合的末端增加删除元素时，使用ArrayList或Vector；当对数据的操作为指定位置的插入或删除时，使用LinkedList；当在多线程中使用容器时，选择Vector更好。
