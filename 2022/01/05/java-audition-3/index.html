2、面向对象技术

2.1、面向对象与面向过程的区别

具体而言，两者之间主要有以下区别：

  • 出发点不同。面向对象强调把问题域的要领直接映射到对象及对象之间的接口上，而面向过程强调的是过程的抽象画与模块化。
  • 层次逻辑不同关系不同。
  • 数据处理方式与程序控制方式不同。
  • 分析设计与编码转换方式不同。

2.2、面向对象的特征

面向对象的主要特征包括抽象、继承、封装和多态。

  • 抽象。忽略与当前目标无关的以便更充分的注意与当前目标有关的方面。分为过程抽象和数据抽象。
  • 继承。是一种联结类的层次模型，并允许和鼓励类的重用，提供了一种明确表述共性的方法。
  • 封装。指将客观事物抽象成类，每个类对自己的数据和方法实行保护。
  • 多态。指允许不同类的对象对同一消息做出相应。包括参数化多态和包含多态。

2.3、面向对象的优点

主要有以下优点：

  • 较高的开发效率。
  • 保证软件的鲁棒性。
  • 保证软件的高度可维护性。

2.4、组合与继承

通过继承，子类可以使用父类中的一些成员变量与方法，从而提高代码的复用性，提高开发效率。继承主要有以下特性：

  • Java中不支持多重继承。子类之多只能有一个父类，但是可以通过实现多个接口达到同样的效果。
  • 子类只能继承父类的非私有（public和protected）成员变量和方法。
  • 当子类中定义的成员变量和父类中定义的成员变量同名时，子类中的成员变量会覆盖父类的成员变量，而不会继承。
  • 当子类中的方法与父类中的方法有相同的函数签名（相同的方法名，相同的参数个数与类型）时，子类会覆盖父类的方法，而不会继承。

2.5、多态的实现机制（重载和覆盖）

Java中，多态主要有两种表现方式：

  • 方法的重载（overload）：指同一个类中有多个同名的方法，但这些方法都有着不同的参数，是一种编译时多态。
  • 方法的覆盖（override）：子类可以覆盖父类的方法，这样同样的方法在子类和父类中有着不同的表现形式。是一种运行时多态。

两者又有什么区别呢？具体而言，有以下区别：

  • 覆盖是子类和父类之间的关系，是垂直关系；重载是同一个类中方法之间的关系，是水平关系。
  • 覆盖只能由一个方法或只能由一对方法产生关系；重载是多个方法之间的关系。
  • 覆盖要求参数列表相同；重载要求参数列表不同。
  • 覆盖关系中，调用方法体是根据对象的类型来决定；重载关系是根据调用时的实参表与形参表来选择方法体的。

2.6、抽象类与接口的异同

两者的相同点如下：

  • 都不能被实例化。
  • 接口的实现类或抽象类的子类都只有实现了接口或抽象类中的方法后才能被实例化。

两者的不同点如下：

  • 接口需要实现（用implements），而抽象类需要继承（用extends），一个类可以实现多个接口，但只能继承一个抽象类。
  • 接口强调特定功能的实现，而抽象类强调所属关系。
  • 接口中定义的成员变量默认为public static final，只能够有静态的不能被修改的数据成员，且必须赋初值；而抽象类可以有自己的数据成员变量，也可以有非抽象的成员方法。当功能需要累积时用抽象类，不需要累积时用接口。
  • 接口被运用于实现比较常用的功能，便于日后维护或添加删除方法；而抽象类倾向于充当公共类的角色，不适用于日后重新修改。
  • 接口可以继承接口，抽象类可以实现接口，抽象类也可以继承具体类，也可以有静态的main方法。

他们的区别主要如下：

  • 静态内部类不依赖于外部类实例而被实例化，不能与外部类有相同的名字，只能访问外部类的静态成员和静态方法（包括private类型）。
  • 成员内部类只有在外部类实例化后才能被实例化，可以自由引用外部类的属性和方法，不可以定义静态的属相和方法，类中不能有静态成员。
  • 局部内部类不能被public、protected、private和static修饰，只能访问方法中定义为final的局部变量，作用范围也只是其所在的代码块，因此很少使用。
  • 匿名内部类是没有类名的，不使用class、extends、implements，无构造函数，必须继承一个父类或实现一个接口，不能定义静态成员、方法或类，不能被public、static、protected、private修饰，一定实在new的后面，且只能创建一个实例。
