4、基本类型与运算（上）

4.1、基本数据类型

Java中一共有8种原始数据类型（byte、short、int、long、float、double、char、boolean），在声明后会立刻在栈上被分配内存空间。除此之外，其他类型都是引用类型，在声明时不会被分配内存空间，只是存储了一个内存地址（类似于指针）。基本数据类型对比如下表：

数据类型 字节长度 范围 默认值 包装类
int 4 [-231,231-1] 0 Integer
short 2 [-32768,32767] 0 Short
long 8 [-263,263-1] 0L或0l Long
byte 1 [-128,127] 0 Byte
float 4 32位IEEE754单精度范围 0.0F或0.0f Float
double 8 64位IEEE754双精度范围 0.0 Double
char 2 Unicode [0,65535] u0000 Character
boolean 1 true和false false Boolean

封装类型与原始类型有许多不同点：首先原始数据类型在传递参数时都是按值传递，而封装类型都是按引用传递；其次，当封装类型和原始类型用作某个类的实例数据时，它们所指定的默认值不同，对象引用实例变量的默认值为null，而原始类型实例变量与它们的类型有关。
除此以外，Java中，默认声明的小数是double类型的，直接写的整型数字是int类型的。因此在对float类型的变量进行初始化时需要进行类型转换。

引申：Jav中的null值是什么？在内存中null又是什么？
null仅仅表明该引用目前没有指向任何对象，即将引用变量的值全部置0。String x = null表示定义了一个变量x，x中存放的是String引用，此处为null。

4.2、不可变类

不可变类是指当创建了这个类的实例后，就不允许修改它的值了。即只允许别的程序读，不允许别的程序修改。创建不可变类的原则：

  • 类中所有成员变量被private修饰；
  • 类中没有写或者修改成员变量的方法，只提供了构造函数；
  • 确保类中所有方法不会被子类覆盖；
  • 如果一个类成员不是不可变量，那么在成员初始化或者使用get方法获取时，需要通过clone方法确保类的不可变性；
  • 如果有必要，可以使用覆盖Object类的equals()方法和hashCode()方法。

4.3、值传递和引用传递

Java提供了两种参数传递的方式：值传递和引用传递。

  • 值传递：在方法调用中，实参会把它的值传递给形参，形参只是用实参的值初始化一个临时的存储单元，因此形参与实参虽然有着相同的值，但是存储单元不同，因此对形参的改变不会影响实参的值。
  • 引用传递：在方法调用中，传递的是对象（对象的地址），这是形参和实参的对象指向同一块存储单元，因此对形参的修改就会影响实参的值。

4.4、数据类型转换

Java中，当参与运算的两个变量的数据类型不同时，就需要进行隐式的数据类型转换，转换规则为：从低精度向高精度转换，即优先级满足：byte<short<char<int<long<float<double。类型转换可以分为以下两种：

  • 类型自动转换：低级数据类型可以自动转换为高级数据类型。如下表所示：
操作数1的类型 操作数2的类型 转换后的类型
long byte short char int long
int byte short chat int
float byte short int char long float
double byte short int char long float double

当类型自动转换时需要注意：
(1) char类型的数据转换为高级类型，会转换成对应的ASCII码；
(2) byte、char、short类型的数据在参与运算时会自动转换为int类型，但当使用个“+=”运算时不会产生类型转换；
(3)基本数据类型与boolean类型是不能相互转换的。

  • 强制类型转换：当需要从高级数据类型转换为低级数据类型时，需要进行强制类型转换。如下表所示：
源操作数的类型 转换后操作数的类型
byte char
char byte char
short byte char
int byte char short
long byte char short int
float byte char short int long
double byte char short int long float

需要注意的是，强制类型转换可能会损失精度。
byte、short和char类型在运算时会强制转换为int类型再进行计算，如果需要得到short类型的结果，就必须显式地把运算结果转换为short类型，例如：short s1 = 1;s1=(short)(s1+1),否则会编译错误；还有一种情况，short s1=1;s1+=1就可以编译通过。
